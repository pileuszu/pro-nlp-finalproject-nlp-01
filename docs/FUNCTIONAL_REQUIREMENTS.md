# 기능 상세 명세서 (Detailed Functional Requirements)

이 문서는 모두취업 플랫폼의 주요 기능들을 상세하게 정의합니다.

---

## 1. 인증 및 계정 관리 (Authentication & Account)

### F-AUTH-01: 카카오 로그인
- **기능 설명**: 카카오 계정을 연동하여 별도의 회원가입 절차 없이 쉽고 빠르게 시스템에 접속한다.
- **API**: `GET /api/auth/kakao/callback`

---

## 2. 포트폴리오 관리 (Portfolio Management)

### F-PORT-01: 외부 소스 데이터 수집 및 등록
- **기능 설명**: GitHub, Notion, PDF 파일 등을 등록하면 **비동기 백그라운드 작업**으로 텍스트를 추출하고 분석한다.
- **입력값**: 소스 타입(URL/File), 제목
- **출력값**: 생성된 포트폴리오 ID (상태: PENDING)
- **API**: 
    - `POST /api/portfolios/upload` (File)
    - `POST /api/portfolios/notion` (Notion)
    - `POST /api/portfolios/github` (GitHub)

### F-PORT-02: 실시간 미리보기 (Preview)
- **기능 설명**: 등록 전, URL이나 파일을 분석하여 추출될 내용을 미리 확인한다. (저장되지 않음)
- **API**: `POST /api/portfolios/analyze`

### F-PORT-03: AI 프로젝트 분석 (Background Job)
- **기능 설명**: 수집된 텍스트를 AI가 분석하여 프로젝트명, 역할, 기술 스택, 성과 등을 구조화하고 임베딩(Vector)을 생성한다.
- **트리거**: 포트폴리오 생성 직후 자동 실행 (Cloud Run Jobs)

### F-PORT-04: 포트폴리오 관리 (CRUD)
- **기능 설명**: 분석된 내용을 수정하거나 프로젝트를 삭제한다.
- **API**: `GET`, `PATCH`, `DELETE /api/portfolios/{id}`

---

## 3. 채용 공고 관리 (Recruit & Recommendation)

### F-RECR-01: 공고 필터링 및 검색
- **기능 설명**: 기술 스택, 직무, 카테고리 등으로 공고를 검색한다.
- **API**: `GET /api/recruits`

### F-RECR-02: AI 맞춤형 공고 추천
- **기능 설명**: 사용자의 포트폴리오 임베딩과 공고의 JD 임베딩 간 유사도를 분석하여 추천 순위를 제공한다.
- **API**: `GET /api/recruits/recommend`

---

## 4. 자기소개서 지원 (Cover Letter Support)

### F-LTR-01: AI 자기소개서 초안 생성
- **기능 설명**: 공고와 내 프로젝트 경험을 결합하여 자소서 초안을 생성한다.
- **API**: `POST /api/cover-letters/generate`

### F-LTR-02: 자소서 관리
- **기능 설명**: 생성된 자소서를 저장, 조회, 수정한다.
- **API**: `GET`, `PATCH /api/cover-letters/{id}`

---

## 5. 비기능 요구사항
- **성능**: 헤비한 AI 작업(분석/임베딩)은 **비동기 큐(Jobs)**로 처리하여 사용자 경험을 저해하지 않는다.
- **확장성**: Backend(API)와 Worker(Jobs)를 분리하여 독립적으로 스케일링한다.

## 1. 인증 및 계정 관리 (Authentication & Account)

### F-AUTH-01: 카카오 로그인
- **기능명**: 카카오 소셜 로그인
- **기능 설명**: 카카오 계정을 연동하여 별도의 회원가입 절차 없이 쉽고 빠르게 시스템에 접속한다.
- **트리거**: 로그인 페이지에서 '카카오 로그인' 버튼을 클릭했을 때
- **입력값**: 
    - 카카오 인증 코드 (Auth Code, Callback을 통해 수신)
- **출력값**: 
    - 성공 여부
    - Access Token (JWT)
    - Refresh Token
    - 사용자 기본 정보 (이름, 카카오 프로필 이미지, 이메일 등)
- **처리 로직**: 
    1. 카카오 인증 서버로 Access Token 요청 (인가 코드 사용)
    2. 획득한 토큰으로 카카오 사용자 정보 조회 (이메일, 닉네임, 프로필 사진 등)
    3. 기존 가입 여부 확인 (신규 사용자일 경우 자동 회원가입 및 초기 프로필 생성)
    4. 서비스 전용 권한 확인 및 JWT 토큰 발급
    5. 세션 유지 및 메인 대시보드로 리다이렉트
- **예외 처리**: 
    - 사용자가 인증 취소: "로그인이 취소되었습니다" 메시지와 함께 로그인 페이지 유지
    - 필수 항목(이메일 등) 동의 거부: 서비스 이용을 위한 필수 항목 동의 재요청 안내
    - 카카오 API 서버 장애: "카카오 로그인 서비스를 일시적으로 사용할 수 없습니다" 안내
- **API**: `GET /api/auth/kakao/callback`

## 2. 포트폴리오 관리 (Portfolio Management)

### F-PORT-01: 외부 소스 데이터 수집
- **기능명**: 포트폴리오 데이터 추출
- **기능 설명**: GitHub, 기술 블로그 URL 또는 PDF 파일을 통해 프로젝트 데이터를 수집한다.
- **트리거**: 사용자가 포트폴리오 추가 페이지에서 '수집 시작' 버튼을 클릭했을 때
- **입력값**: 
    - 소스 타입 (URL 또는 FILE)
    - 데이터 (GitHub URL, 블로그 URL, 또는 PDF 파일)
- **출력값**: 
    - 추출된 텍스트 데이터 요약
    - 분석 대기 상태 메시지
- **처리 로직**: 
    1. 입력된 URL 유효성 또는 파일 형식 검증
    2. Crawler(URL) 또는 Parser(PDF)를 통한 텍스트 추출
    3. 데이터 정제 (HTML 태그 제거, 불필요한 공백 제거)
    4. 추출된 데이터를 AI 분석 엔진으로 전달
- **예외 처리**: 
    - 유효하지 않은 URL: "올바른 URL 형식을 입력해주세요"
    - 접근 불가능한 Private Repo: "접근 권한이 없는 레포지토리입니다"
    - 손상된 PDF 파일: "파일을 읽을 수 없습니다"
- **API**: `POST /api/portfolios/extract`

### F-PORT-02: AI 프로젝트 분석 및 정제
- **기능명**: LLM 기반 프로젝트 분석
- **기능 설명**: 수집된 비정형 텍스트를 분석하여 구조화된 프로젝트 데이터를 생성한다.
- **트리거**: 데이터 수집이 완료된 직후 배경 프로세스로 자동 실행
- **입력값**: 정제된 텍스트 뭉치
- **출력값**: 
    - 프로젝트명, 역할, 사용 기술(Stack), 핵심 성과, 기간
- **처리 로직**: 
    1. LLM(임베딩 모델)을 통한 문맥 분석
    2. 기술 스택 키워드 추출 및 표준화
    3. 성과 지표(숫자 등) 중심의 요약 생성
    4. 결과를 사용자 검토 가능 상태로 저장
- **예외 처리**: 
    - 텍스트 정보 부족: "분석할 수 있는 정보가 충분하지 않습니다"
    - AI 모델 응답 지연: 타임아웃 처리 및 재시도 큐 등록
- **API**: `POST /api/portfolios/analyze`

### F-PORT-03: 포트폴리오 관리 (CRUD)
- **기능명**: 프로젝트 상세 관리
- **기능 설명**: AI가 분석한 내용을 수정하거나 사용자가 직접 프로젝트를 추가/삭제한다.
- **트리거**: 사용자가 대시보드에서 편집 버튼을 클릭하거나 저장 시
- **입력값**: 프로젝트 개별 필드 정보 (제목, 기술, 내용 등)
- **출력값**: 저장 결과 상세 데이터
- **처리 로직**: 
    1. 변경사항 유효성 검사
    2. 데이터베이스 Row 업데이트 또는 생성
    3. 대표 포트폴리오 설정 시 가중치 반영
- **API**: `PATCH /api/portfolios/{id}`, `DELETE /api/portfolios/{id}`

## 3. 채용 공고 관리 (Recruit & Recommendation)

### F-RECR-01: 공고 필터링 및 검색
- **기능명**: 조건별 공고 탐색
- **기능 설명**: 기술 스택, 직무, 지역 등 필터를 적용하여 채용 공고를 검색한다.
- **트리거**: 사용자가 검색창에 입력하거나 필터 옵션을 변경했을 때
- **입력값**: 키워드, 직무 태그 리스트, 정렬 조건
- **출력값**: 필터링된 공고 목록 (Pagination 적용)
- **처리 로직**: 
    1. 검색 쿼리 정규화
    2. 데이터베이스 Index를 활용한 필터링 쿼리 수행
    3. 실시간 인기순/최신순 정렬 적용
- **API**: `GET /api/recruits`

### F-RECR-02: AI 맞춤형 공고 추천
- **기능명**: 내 역량 매칭 공고 추천
- **기능 설명**: 사용자의 포트폴리오와 공고의 유사도를 분석하여 최적의 공고를 상단에 배치한다.
- **트리거**: 사용자가 채용 페이지에 진입했을 때
- **입력값**: 사용자 포트폴리오 데이터셋 ID
- **출력값**: 매칭 점수(Score)가 포함된 추천 공고 리스트
- **처리 로직**: 
    1. 사용자 포트폴리오 역량 벡터 추출
    2. 전체 공고 JD(Job Description) 벡터와의 코사인 유사도 연산
    3. 기술 스택 일치 항목에 대한 가중치 추가 점수 부여
    4. 상위 매칭 공고 정렬 반환
- **API**: `GET /api/recruits/recommendations`

## 4. 자기소개서 지원 (Cover Letter Support)

### F-LTR-01: AI 자기소개서 초안 생성
- **기능명**: 포트폴리오 기반 자소서 생성
- **기능 설명**: 선택한 프로젝트 경험을 활용하여 공고 문항에 최적화된 자소서 초안을 작성한다.
- **트리거**: 사용자가 'AI 초안 생성' 버튼을 클릭했을 때
- **입력값**: 
    - 대상 공고 ID
    - 활용할 프로젝트 ID 리스트
    - 작문 스타일 (전문적인/친근한 등)
- **출력값**: 생성된 자기소개서 텍스트
- **처리 로직**: 
    1. 프로젝트 상세 내용과 공고 요구사항 결합
    2. LLM 프롬프트 생성 (경험-역량 매칭 중심)
    3. 문장 생성 및 가독성 최적화
    4. 생성된 서류를 '작성 중'인 문서로 저장
- **예외 처리**: 
    - 입력 문항 없음: "문항 내용을 입력해주세요"
- **API**: `POST /api/cover-letters/generate`

### F-LTR-02: 실시간 AI 첨삭 및 교정
- **기능명**: 실시간 문맥 편집 지원
- **기능 설명**: 사용자가 작성 중인 자소서의 문맥을 분석하여 더 나은 표현이나 기술 역량을 강조할 문장을 추천한다.
- **트리거**: 사용자가 에디터에서 특정 문장을 선택하거나 '첨삭 요청' 시
- **입력값**: 작성 중인 텍스트 블록
- **출력값**: 제안된 수정 문장 리스트, 오타 교정 결과
- **처리 로직**: 
    1. 문장 단위 형태소 및 문맥 분석
    2. 전문 용어 사용 적절성 검사
    3. 더 임팩트 있는 성과 중심 문장으로 재구성 제안
- **API**: `POST /api/cover-letters/refine`

## 5. 비기능 요구사항 (Non-Functional Requirements)

- **성능**: AI 분석 API 응답은 5초 이내 완료를 지향하며, 지연 시 프로그레스바 또는 비동기 알림을 제공한다.
- **보안**: 모든 포트폴리오 데이터는 암호화되어 관리되며, 소유자 본인 외에는 접근할 수 없다.
- **가용성**: 채용 시즌 트래픽 급증에 대비하여 공고 검색 API는 캐싱 레이어(Redis 등)를 거친다.
