# 프론트엔드 모킹 전략 (Mocking Strategy)

백엔드 개발과 병행하여 효율적인 프론트엔드 개발 및 테스트를 진행하기 위해 **MSW (Mock Service Worker)**를 적극 활용합니다.

## 1. MSW (Mock Service Worker) 도입

서비스 워커를 사용하여 네트워크 요청을 가로채고, 실제 서버가 응답하는 것처럼 모의 데이터(Mock Data)를 제공합니다.

### 주요 장점
- **코드 무결성**: 실제 API 호출 코드(`fetch`)를 수정할 필요 없이 네트워크 레벨에서 작동합니다.
- **다양한 상황 시뮬레이션**: 성공, 실패(400, 500), 네트워크 지연(delay) 등을 자유롭게 설정하여 UI 대응력을 높입니다.
- **실제 백엔드 전환 용이**: 백엔드 준비 완료 시 MSW 설정만 끄면 즉시 실제 서버와 연동됩니다.

## 2. 구현 방식

### 핸들러 관리 (`src/mocks/handlers.ts`)
모든 API 엔드포인트별 모킹 로직은 `handlers.ts`에서 관리합니다. 현재 다음과 같은 주요 기능들이 모킹되어 있습니다.
- **포트폴리오 분석**: GitHub/Notion 데이터를 기반으로 한 AI 분석 결과 생성.
- **자기소개서 관리**: 목록 조회, 저장, 삭제 기능.
- **채용 공고**: 업계 실제 공고 데이터 기반의 목록 제공.

### 서비스 워커 설정
- 브라우저 환경에서는 `src/mocks/browser.ts`를 통해 런타임에 서비스 워커를 시작합니다.
- 개발 환경(`dev`)에서만 활성화되도록 설정되어 있습니다.

## 3. 워크플로우

1.  **API 명세 합의**: 백엔드 팀과 데이터 구조를 먼저 확정합니다.
2.  **모킹 데이터 구축**: 합의된 구조를 바탕으로 `handlers.ts`에 응답 로직을 작성합니다.
3.  **UI 개발**: 실제 데이터가 오는 것처럼 프론트엔드 로직을 완성합니다.
4.  **연동 테스트**: 백엔드 배포 후 MSW를 비활성화하여 실제 통신 상태를 확인합니다.
